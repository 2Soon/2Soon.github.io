# 什么是交互设计

交互设计是指人工制品、环境和的行为，以及传达这种的外形元素的设计与。不像传统的设计学科主要关注形式，则是关注内容和内涵，而交互设计首先旨在和描述事物的行为方式，然后描述传达这种行为的最有效形式。

首先要知道什么是交互

交互，及沟通交流，发生互动关系。比如人和人之间的交互就比较好理解，最经典的一幕可以用孙悟空智斗金角大王、银角大王。金角大王说：孙行者，我叫你一声你敢答应吗？然后，金角大王就叫：孙行者。孙悟空回答：爷爷在此。就这样，孙悟空就被收进去了。这就是一个简单的交互。

再比如，我们每天上班，到公司和同事打招呼。你说：“早上好呀”，同事回答“早”。这也是一个常见的交流互动。

交互设计的原则

  • 可视化
  • 反馈
  • 限制
  • 自然匹配
  • 一致性
  • 可示的 (启发性：充分准确的操作提示。)

一些点，你看下能不能匹配到那个原则上

水龙头

而这款水龙头的另一大特色，也在于这款精华的木板之上，它不仅是一个开关，更是一个“杯托”，将水杯置于木板之上，我们不用再接水的过程中一直拿着水杯，水杯放置其上便可自然去睡，抬起水杯则自然关闭，而木板的设计，也体现了设计师注重细节的态度，还原原木元素，铜质原料转印木质纹理，不仅保证了它的寿命，更让它在视觉上为人们带来享受。

通常来说，使用水龙头时少不了旋拧开关的动作，而贝朗却把杠杆原理运用到产品设计当中，让杯子只需要像是坐跷跷板式的放上平板，既有饮用水流出;而一旦拿走杯子，龙头便会自动关闭。这样的设计让Hand-free Cup Filler不仅看起来非常直观、简洁，而且也把喝水这件事变得生动有趣起来，让人忍不住多喝几次水。

# 基本用法

远程仓库尽量与本地保持一致

clone

在对应文件夹下，git bash here

git clone https://gitee.com/xianghao98/vmail2.git

本地仓库

  1. git status

  1. git status -s使输出信息更简洁

  1. 将未跟踪状态的文件转为已跟踪状态 git add tt.txt

  1. 将暂存区的文件取消暂存 git reset

要想提交的文件，必须先提交到暂存区。

  1. git commit -m “update README.md”

    若不加 -m 则自动打开文件，输入i编辑，：wq退出

  1. git rm tt.txt 此时只是在工作区删除，需要commit -t提交才是删除

  2. 创建文件 .gitignore ，忽略文件

  3. git log查看日志记录

远程仓库

  1. 查看

  2. 添加远程仓库

  3. 从远程克隆

    git clone

    不仅仅克隆文件，还会有日志信息、历史记录等

  4. 移除无效仓库

    git remote rm

    只是从本地移除远程仓库，并不会真正影响到远程仓库

  5. 从远程仓库抓取与拉取

    git fetch 是从远程仓库获取最新版本到本地仓库，不会自动merge

git pull 是从远程仓库获取最新版本并merge到本地仓库

  1. 推送到远程仓库

    git push origin master

    需要输入密码

开发分支

git init命令默认创建master分支

  1. 查看分支

    列出所有本地分支

    git branch

    列出所有远程分支

    git branch -r

    列出所i有本地分支和远程分支

    git branch -a

  1. 创建分支

    git branch b1

  2. 切换分支

    git checkout b1

  3. 推送至远程仓库分支

    git push origin b1

  4. 合并分支，合并之后为master

    在master下 git merge b1

  1. 删除分支

git标签

发布关键版本时，打标签

  1. 列出所有标签

    git tag

  2. 创建新标签

    git tag [tagName]

  1. 查看tag信息

    git show [tag]

  2. 将标签推送至远程仓库

    git push [remote] [tag]

  3. 检出标签

    新建一个分支，指向某个tag

    git checkout -b [branch] [tag]

  4. 删除标签

    git tag -d [tag] 删除本地标签

    git push origin :refs/tags/[tag]
